# Cheok-Chew (척추 밈 제스처 인식기) 척·츄

## 🚀 프로젝트 개요 (Overview)

**Cheok-Chew**는 브라우저 웹캠으로 사용자의 **주먹(Fist)** 제스처를 실시간으로 인식하여 양손에 각각 '척!', '츄!' 텍스트와 음향 효과를 출력하는 인터랙티브 웹 장난감 프로젝트입니다. 
사용자의 손 궤적에 따라 실시간으로 구부러지고 휘어지는 탄성 있는 "척추" 모델이 중앙 캔버스에 렌더링되며, 제스처의 상하 위치(Y축)에 따라 음성 효과의 피치(Pitch)와 속도가 변주되어 우스꽝스럽고 찰진 상호작용을 경험하실 수 있습니다.

---

## 💡 개발 동기와 배포 방식의 진화 (.exe 보안 차단 이슈 극복)

본 프로젝트는 초기에 **Python + OpenCV + MediaPipe** 기반의 데스크탑 애플리케이션으로 기획되었으며, 이를 **PyInstaller**를 통해 `.exe` 싱글 실행 파일로 패키징하여 배포하려 했습니다.
하지만 **Windows OS가 자체적인 보안 정책(Windows Defender, SmartScreen 등)으로 인해 서명되지 않은 개인의 `.exe` 바이너리 실행을 맬웨어로 간주해 강제로 차단해버리는 치명적인 배포 이슈**가 발생했습니다.

친구들에게 링크 하나만 던져주면 타사 PC나 스마트폰의 환경 제약 없이 누구나 즉시 밈을 켜서 즐길 수 있게 하겠다는 원래의 목표를 달성하기 위해, **100% 프론트엔드 기반의 Web App 형태(`index.html` 단일 파일)** 로 코어 로직을 전면 재설계하고 포팅하는 완전히 새로운 배포 방식을 고안하게 되었습니다.

## 🛠 주요 기술 스택과 응용 기법들 (Technologies & Techniques)

1. **Google MediaPipe Tasks Vision JS**
   - 브라우저 환경에서 무거운 OpenCV 엔진을 대체하는 최경량 제스처 및 랜드마크 트래킹 코어.
   - 단일 싱글톤 객체로 캐싱 관리하여 카메라 프레임 탐색마다 발생하는 모델 추론 부하 리소스를 극단적으로 감축했습니다.

2. **Web Audio API & Base64 Payload Injection**
   - 기존 HTML `<audio>` 태그의 물리적 한계로, 연속적인 다중 제스처 타격 시 음원의 오버래핑(Overlapping)이 불가능하고 뚝뚝 끊기는 지연 현상이 컸습니다.
   - **기법**: `AudioContext` 메모리 버퍼 캐싱. 이벤트마다 새 소스 노드를 생성하여 딜레이 없이 수십 개의 소리가 겹쳐 나도록 설계했습니다.
   - **Base64** 캡슐화: 독립적인 HTML 앱으로서의 배포율을 높이기 위해 서버 없이 구동 시 발생하는 로컬 오디오 404 Fetch 에러를 분석하고, 약 100KB 분량의 mp3 에셋들을 압축 해제 없이 순수 바이너리 텍스트(Base64) 스트링으로 변환해 소스 코드 최상단에 영구 주입(Injection)했습니다. 이를 통해 로컬 파일인 `index.html` 더블클릭만으로 완벽하게 모든 동작이 수행되는 인캡슐레이션을 달성했습니다.

3. **HTML5 Canvas Spring Physics**
   - 일반적인 CSS 트랜지션 애니메이션으로는 손 추적이 일시적으로 튀거나 끊길 때 좌표계 자체가 뒤틀리는 현상이 있었습니다.
   - **기법**: 게임엔진에서 착안한 후크의 법칙(Hooke's Law: `F = -kx`) 댐핑 스프링 물리 함수를 작성하고, 브라우저 렌더 큐(`requestAnimationFrame`) 루프로 캔버스에 직접 척추 각도 베지어 곡선(Bezier Curve)을 스케치하여 쫀득쫀득한 반동 에니메이션 체계를 완성했습니다.

## 🧗‍♂️ 시행착오와 문제 해결 과정 (Trial & Errors)

### 1. Windows 보안 모델의 장벽과 웹으로의 대대적 포팅
- **문제**: 정성껏 만든 데스크탑 프로그램을 `.exe`로 패키징해 배포했으나, "Windows가 PC를 보호했습니다 (알 수 없는 게시자)" 등의 무시막지한 경고창으로 인해 실제 유입 사용자들이 프로그램을 구동조차 하지 못하는 심각한 접근성 문제가 도출되었습니다.
- **해결**: 사용자 로컬 저장소 의존성을 버리고 브라우저 샌드박스로 구동 환경을 이송시켰습니다. MediaPipe JS 라이브러리를 통해 파이썬 기반 추론 로직을 자바스크립트에 성공적으로 이식하여 설치의 번거로움 자체를 완전히 소멸시켰습니다.

### 2. 절대 랜드마크 스케일링 (Scaling)의 엇갈림
- **문제**: 브라우저 기반의 Vision AI 모델은 원본 비디오 해상도의 상대적 비율(0.0~1.0) 좌표를 제공합니다. 켄버스가 브라우저 창 100vw, 100vh로 반응형 지정되어 있을 때 반환 마스크의 좌표계가 브라우저 DOM 마우스 좌표계와 동기화되지 않아 그래픽이 공중에 붕 뜨는 오차가 발생했습니다.
- **해결**: 캔버스의 내부 렌더링 해상도 속성을 `window.innerWidth/innerHeight`와 강제로 일치화(Sync)하여 정적 픽셀 비율 지수를 얻고, 각 랜드마크 튜플에 해당 실수 계수를 곱해 정확한 절대 좌표 스케일링 알고리즘 로직을 구현했습니다.

### 3. 깜빡임 (Flickering) 현상 및 이종 렌더링 루프 동기화
- **문제**: 손 랜드마크가 트래킹 영역을 아주 찰나에 벗어난 프레임 직후, 그래픽 척추 객체가 즉각 파괴되면서 화면이 지속적으로 점멸하는 깜빡임 증상이 나타났습니다.
- **해결**: 객체 지향 상태 패턴(State Pattern)을 적용하여, 이전 프레임의 추론 랜드마크 위치를 내부 캐시 메모리에 보존했습니다. 추적 좌표가 완전히 유실되더라도 하드킬 처리하지 않고 지정 렌더링 생존 프레임 틱(Tick) 동안 스프링 관성을 유지시켜 자연스러운 페이드아웃 효과로 해결했습니다.

### 4. 모바일 Safari 등 Webkit 엔진의 엄격한 Audio Autoplay 제약
- **문제**: 일반 데스크탑에서는 잘 작동하던 이펙트 사운드가 모바일 아이폰(iOS) Safari에서 실행 시 스크립트로 동작하는 모든 AudioContext가 시스템 레벨에서 무음 차단(Blocked)되는 정책의 피해를 입었습니다.
- **해결**: 악의적 광고를 제한하는 브라우저 보안 컨텍스트에 우회 대응하기 위해, `AudioContext` 선언부와 `.resume()` 오디오 잠금 해제 체인을 사용자(User)가 UI 화면을 실제로 명시적 터치하는 이벤트('시작하기' 버튼 클릭 콜백) 내부 콜 스택 최하단으로 깊숙이 이동 바인딩시켜 사용자 권한을 자동 위임받고 오디오 락(Lock)을 퍼펙트하게 해제했습니다.

## 🏃 어떻게 실행하나요?

본 레포지토리는 데스크탑 앱 환경을 완벽하게 포팅해 낸 단일-뷰 웹 애플리케이션 리포지토리입니다. Python 환경이나 기타 모듈의 설치가 전혀 필요 없으며 하단 방식 중 택일하시면 됩니다.

- **브라우저에 바로 호스팅하기 (권장):** 프로젝트 루트의 `index.html` 파일을 `GitHub Pages` 기능이나 `Netlify Drop` 등의 무료 정적 배포 플랫폼에 업로드하시면 스마트폰, PC를 가리지 않고 즉시 반응형 서비스로 동작합니다.
- **로컬 테스트:** 에디터(VSCode) 내부의 `Live Server`나 Python `http.server` 등을 통해 실행 후 `localhost`로 접근해 카메라 권한을 허용해주세요. (단축키 클릭으로 `file://` 경로를 통해 브라우저를 열면 크로스 오리진 및 카메라 권한 정책에 의해 일부 센서 로드가 차단될 수 있습니다.)
